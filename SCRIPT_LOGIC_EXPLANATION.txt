PYTHON SCRIPT OVERVIEW - update_footers.py

SCRIPT LOGIC:
=============

1. READ MASTER FOOTER
   ├─ Opens: c:\xampp\htdocs\Webbuilders Projects\A2Z\A2Z\master_footer.txt
   ├─ Reads entire file content (UTF-8 encoding)
   └─ Stores in memory for repeated use

2. FOR EACH HTML FILE (19 total):
   ├─ Check if file exists
   ├─ If exists:
   │  ├─ Read file content (UTF-8 encoding)
   │  ├─ Search for pattern: <div class="btSiteFooter">.*?</div><!-- /btSiteFooter -->
   │  ├─ If pattern found:
   │  │  ├─ Replace entire matched section with master footer content
   │  │  ├─ Write updated content back to file
   │  │  └─ Report: ✓ {filename}: Successfully updated
   │  └─ If pattern NOT found:
   │     └─ Report: ✗ {filename}: btSiteFooter section not found
   └─ If file doesn't exist:
      └─ Report: ✗ {filename}: File not found

3. SUMMARY
   ├─ Count successful updates
   ├─ Count failed updates
   └─ Display final report

REGEX PATTERN EXPLAINED:
========================

Pattern: <div class="btSiteFooter">.*?</div><!-- /btSiteFooter -->

Breaking it down:
  <div class="btSiteFooter">          ← Matches opening div tag
  .*?                                  ← Matches ANY content (non-greedy)
  </div><!-- /btSiteFooter -->        ← Matches closing div with comment

Non-greedy (.*?) means it stops at the FIRST closing </div><!-- /btSiteFooter -->
This ensures we don't accidentally match multiple footers if they exist.

WHY THIS APPROACH WORKS:
========================

✓ Safe: Only modifies the footer section
✓ Reliable: Uses regex to find exact boundaries
✓ Non-destructive: Leaves everything else in the file intact
✓ Reversible: Can easily revert with git
✓ Efficient: Processes all files in < 5 seconds
✓ Reporting: Shows exactly what happened to each file

PERFORMANCE:
=============

Expected timing per file:
  - Reading: ~0.1 ms
  - Regex matching: ~0.1 ms
  - Replacement: ~0.1 ms
  - Writing: ~0.5 ms
  - Total: ~1 ms per file

Total for 19 files: ~20 ms (plus console output time)

=============

ERROR HANDLING:
===============

The script handles:
  ✓ Missing master footer file
  ✓ Missing HTML files
  ✓ File read errors
  ✓ File write errors
  ✓ Encoding issues
  ✓ Regex matching failures

For each error, the script:
  1. Prints an error message with filename
  2. Increments the failure counter
  3. CONTINUES to process remaining files (doesn't stop)

This ensures maximum completion even if some files have issues.

IDEMPOTENCY:
============

The script is IDEMPOTENT - it's safe to run multiple times:
  - First run: Updates footer from old to new ✓
  - Second run: Updates footer from new to new (no visible change) ✓
  - Any run: Same result ✓

This is because:
  1. It searches for the exact footer boundaries
  2. It replaces with the exact master footer content
  3. Multiple replacements with same content = same result

UNICODE HANDLING:

The script properly handles:
  ✓ UTF-8 encoded HTML files

  ✓ Accented letters and symbols

By using encoding='utf-8' on all file operations.
